# ボクセルキャラクター制作パイプライン ロードマップ

## 概要

現在のAI依存パイプラインを、ユーザーが自力でキャラクターを制作できるシステムに進化させる。

---

## 現状の課題

| 課題 | 詳細 |
|---|---|
| AI依存 | パラメータ調整、スクリプト作成、トラブル対応をAIが担当 |
| 後処理が多い | 髪: 6ステップ、衣装: 膨張+シフト+干渉修正 |
| モデル別スクリプト | `_de.js` のようにモデルごとにスクリプトを複製 |
| 左右非対称 | ボクセル化時に対称性が保証されない |
| Tポーズ未強制 | マニュアルに記載のみ、自動検出/変換なし |
| パーツ分類が手動 | どのパーツが髪/衣装/装飾品かをAIが判断 |
| 体型依存 | 衣装が特定のBody形状にハードコードで合わせられている |

---

## フェーズ構成

```
Phase 1: FBXインポート & パーツ分類UI
Phase 2: テンプレートシステム（髪キャップ・衣装ベース）
Phase 3: 対称性強制 & Tポーズ保証
Phase 4: 体型適応フィッティング
Phase 5: リアルタイムパラメータ調整UI
```

## Phase 1: FBXインポート & パーツ分類UI

### 目的
3Dモデルを読み込み、画面上でパーツを確認しながらテンプレートカテゴリを割り当てる。

### 機能
1. **FBX/GLB読み込み** — Babylon.jsでブラウザ内に3Dモデルを表示
2. **パーツ一覧表示** — メッシュ名ごとにリスト化
3. **クリック選択** — 3Dビュー上でパーツをクリック → ハイライト表示
4. **カテゴリ割当** — 各パーツにテンプレートカテゴリを選択:
   - Body（体） — ボクセル化対象外（共通Body使用）
   - 髪 → ヘアキャップテンプレート
   - シャツ/ジャケット → 上半身衣装テンプレート
   - ズボン/スカート → 下半身衣装テンプレート
   - ブーツ/靴 → 足元テンプレート
   - 手袋 → 手テンプレート
   - 装飾品（ネクタイ、ベルト等） → 自由形状
   - 除外 — ボクセル化しない
5. **設定保存** — 分類結果を `import-config.json` として保存

### 実装箇所
- 新規ページ: `src/app/model-import/page.tsx`
- Babylon.js の `SceneLoader.ImportMesh` でFBX/GLB読み込み
- メッシュ単位の `ActionManager` でクリック検出

### 出力
```json
{
  "modelFile": "NewCharacter.fbx",
  "parts": {
    "body": { "category": "body", "template": null },
    "hair_main": { "category": "hair", "template": "hair_cap" },
    "jacket": { "category": "upper_body", "template": "shirt" },
    "pants": { "category": "lower_body", "template": "pants" },
    "boots": { "category": "footwear", "template": "boots" },
    "necklace": { "category": "accessory", "template": "free_form" }
  },
  "tPoseValidated": true
}
```

### 完了条件
- [x] FBX/GLBファイルをドラッグ&ドロップで読み込める
- [x] パーツ一覧がリスト表示される
- [x] 3Dビューでパーツクリック → ハイライト
- [x] 各パーツにカテゴリ/テンプレートを割り当てられる
- [x] 設定をJSONとして保存できる
- [x] Delete キーでパーツ非表示
- [x] 不要オブジェクト（ボーン、ヘルパー等）の自動フィルタリング

## Phase 2: テンプレートシステム

### 目的
髪・衣装の基本形状をテンプレートとして用意し、後処理を大幅に削減する。

### テンプレート一覧

| テンプレート名 | 用途 | 基準 |
|---|---|---|
| `hair_cap` | ヘアキャップ（髪の下地） | チビ頭のサイズに合わせた球殻状メッシュ |
| `shirt` | 上半身衣装 | Body上半身の表面+1ボクセル外側 |
| `pants` | 下半身衣装 | Body下半身の表面+1ボクセル外側 |
| `boots` | ブーツ/靴 | Body足部の表面+1ボクセル外側 |
| `gloves` | 手袋 | Body手部の表面+1ボクセル外側 |
| `full_body` | 全身スーツ | Body全体の表面+1ボクセル外側 |

### ヘアキャップの仕組み

```
1. チビ変形済みの頭部形状から球殻を生成
2. 球殻がヘアキャップ（下地）になる
3. 3Dモデルの髪の形状情報（方向・長さ）を取得
4. キャップから外側に向かって「植毛」する
   - 各頭頂ボクセルから法線方向に伸ばす
   - 3Dモデルの髪の長さに応じて長さを決定
   - 色は3Dモデルのテクスチャから取得

結果:
  ┌───┐
 ╱ 髪  ╲    ← 3Dモデルの髪形状に基づく植毛
│ キャップ│   ← チビ頭にフィット済みの下地
│  頭部  │   ← チビ変形済みBody
└───────┘
```

### 衣装テンプレートの仕組み

```
1. Body表面のボクセルを検出（各部位ごと）
2. 表面から法線方向に+1〜+2ボクセル外側にシェルを生成
3. シェルがテンプレート（下地）になる
4. 3Dモデルの衣装形状をテンプレートに投影:
   - テンプレート範囲内 → テンプレートの形状を使用（フィット保証）
   - テンプレート範囲外 → 3Dモデルの形状をそのまま使用（自由形状）
5. 色は3Dモデルのテクスチャから取得

結果: Body表面に確実にフィットし、かつ独自形状も保持
```

### テンプレート + 自由編集の共存

```
テンプレート適用範囲:
  - Body表面に接する部分 → テンプレートが形状を保証
  - Body表面から離れた部分 → 3Dモデルの形状をそのまま使用

例: マント
  - 肩周り → テンプレート（Body肩にフィット）
  - 背中から垂れる部分 → 自由形状（3Dモデルの形状）

例: 非対称アクセサリ
  - テンプレート割当を「free_form（装飾品）」にする
  - → テンプレート適用なし、3Dモデルの形状をそのまま使用
```

### テンプレートファイル形式

```
public/templates/
├── hair_cap.vox          (ヘアキャップ下地)
├── shirt_shell.vox       (上半身シェル)
├── pants_shell.vox       (下半身シェル)
├── boots_shell.vox       (足元シェル)
├── gloves_shell.vox      (手シェル)
└── full_body_shell.vox   (全身シェル)
```

テンプレートは**Body形状から自動生成**する。Body形状が変わればテンプレートも再生成。

### 完了条件
- [x] Bodyからテンプレートシェルを自動生成するスクリプト（`generate_templates.js`）
- [x] ヘアキャップ生成（頭部球殻テンプレート）
- [x] テンプレート + 3Dモデル形状の合成ロジック（ブラウザ上で実装）
- [x] テンプレート範囲外は自由形状として保持
- [x] カテゴリ「accessory」でテンプレートバイパス可能
- [x] model-importページに統合（Classify → Voxelizeの2モード）
- [ ] 後処理（膨張・位置調整）がテンプレート適用済みパーツでは不要

## Phase 3: 対称性強制 & Tポーズ保証

### 目的
左右対称のBodyと衣装を保証し、Tポーズ出力を強制する。

### 3-1: 左右対称化

```
処理フロー:
1. ボクセル化後のデータを読み込み
2. 中心X座標（cx）を計算
3. 左半分（x < cx）のボクセルを基準とする
4. 右半分を削除し、左半分をX軸ミラーコピー
5. 中心線上のボクセルはそのまま保持

オプション: 右半分を基準にするか左半分を基準にするかUI上で選択可能
オプション: 対称化を無効にするチェックボックス（非対称パーツ用）
```

### 3-2: Tポーズ検出・変換

```
Blenderエクスポート時:
1. Armature（ボーン）を検出
2. 両腕のボーン角度を計算
3. Tポーズ判定:
   - 上腕の角度が水平（±10度以内） → OK
   - それ以外 → 警告 + 自動修正オプション
4. 自動修正: UpperArm.rotation を水平に設定してからメッシュ適用

UIでの表示:
  ✅ Tポーズ検出: OK
  ⚠️ Tポーズ検出: Aポーズ（腕角度: -45°）→ [自動修正] ボタン
```

### 完了条件
- [x] ボクセル左右対称化スクリプト（`symmetrize_voxels.js`）
- [ ] 対称化ON/OFFのUI切り替え
- [x] インポートUIにTポーズ検出・状態表示（ボーン角度自動検出）
- [ ] Tポーズ自動修正（Aポーズ→Tポーズ変換 Blenderスクリプト）
- [ ] Blenderスクリプトに Tポーズ検出ロジック追加

## Phase 4: 体型適応フィッティング

### 目的
異なる体型のキャラクターに衣装を自動フィットさせる。

### 仕組み

```
衣装の定義方法（従来）:
  衣装ボクセル = 絶対座標 (x, y, z)

衣装の定義方法（新方式）:
  衣装ボクセル = Body表面からの相対情報
  {
    boneName: "Spine1",          // 最寄りボーン
    surfaceOffset: 1.5,          // Body表面からの距離
    surfaceNormal: [0, -1, 0],   // 表面法線方向
    color: [r, g, b, a]          // 色
  }
```

### 適用フロー

```
1. キャラAのBody + 衣装を読み込み
2. 衣装の各ボクセルについて:
   a. 最寄りのボーン名を特定
   b. Body表面までの距離と方向を計算
   c. 相対情報として記録
3. キャラBのBodyを読み込み
4. キャラBの各ボーン位置・Body表面を基準に:
   a. 相対情報からボクセル位置を再計算
   b. キャラBの体型にフィットした位置に配置

結果:
  キャラA(細身)用のシャツ → キャラB(体格大)に自動フィット
```

### 制限事項
- Body表面から大きく離れたパーツ（マント裾など）はボーンの長さ比でスケーリング
- 非人型ボディには対応しない（人型前提）
- 色は元モデルのテクスチャを維持（体型変化で色は変わらない）

### 完了条件
- [ ] 衣装の相対座標変換スクリプト
- [ ] 体型パラメータの定義（身長、肩幅、胴囲、脚長など）
- [ ] 異なるBody間での衣装自動フィッティング
- [ ] UI上で体型選択 → 衣装プレビュー

## Phase 5: リアルタイムパラメータ調整UI

### 目的
パラメータ調整をスクリプト編集ではなくUI操作で行えるようにする。

### 機能

```
┌──────────────────────────────────────────────┐
│  パーツ調整UI                                 │
│                                              │
│  ┌──────────────┐    調整パラメータ           │
│  │              │                            │
│  │  3Dプレビュー │    位置 X: [0]  ◄──► +1   │
│  │  (リアルタイム│    位置 Y: [-1] ◄──► +1   │
│  │   更新)      │    位置 Z: [-5] ◄──► +1   │
│  │              │                            │
│  └──────────────┘    膨張: [✓] ON            │
│                      膨張量 XZ: [1] ▲▼       │
│                      膨張量 Y:  [2] ▲▼       │
│                                              │
│                      対称化: [✓] ON          │
│                      テンプレート: [シャツ ▼]  │
│                                              │
│                      [適用] [リセット] [保存]  │
└──────────────────────────────────────────────┘
```

### 技術方針
- スライダー変更 → WebWorkerでボクセル変換 → Babylon.jsメッシュ更新
- 変換結果をメモリ上に保持し、保存ボタンで `.vox` ファイルに書き出し
- undo/redo サポート

## 優先順位と依存関係

```
Phase 1 ──→ Phase 2 ──→ Phase 5
  │              │
  │              └──→ Phase 4
  │
  └──→ Phase 3（独立して並行可能）
```

| 順序 | フェーズ | 理由 |
|---|---|---|
| 1st | Phase 1（インポートUI） | 全フェーズの入口。これがないと後続が使いにくい |
| 2nd | Phase 3（対称性・Tポーズ） | 独立実装可能。品質向上に直結 |
| 3rd | Phase 2（テンプレート） | 後処理削減の核心。Phase 1の分類結果を利用 |
| 4th | Phase 5（調整UI） | テンプレートで減った調整作業の残りをUI化 |
| 5th | Phase 4（体型適応） | 複数Body対応。テンプレートシステムの上に構築 |
